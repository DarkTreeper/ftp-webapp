## рҹ“Ӣ Sicher-zu-Committen Pro-Datei

### вң… `pom.xml`
- OK, weil nur Dependency-Referenzen
- Spring Boot, Apache Commons Net,BCrypt вҖ” alle public

### вң… `.env.local.example` und `.env.production.example`
- OK, sind nur TEMPLATES
- Verwenden Placeholder wie `CHANGE_ME_TO_A_LONG_RANDOM_PASSWORD`
- Reine DOKUMENTATION, nicht echte Geheimnisse

### вң… `src/main/resources/application.yml`
- OK, verwendet nur `${...}` Platzhalter
- KEINE hardcodierten Credentials
- Verweist auf Umgebungsvariablen

### вқҢ `.env` (falls present)
- SOFORT aus Git entfernen, wenn dort!
- EnthГӨlt echte FTP-Credentials und App-Passwords
- **Command zum Entfernen:**
  ```bash
  git rm --cached .env
  git commit -m "Remove .env with credentials"
  ```

### вқҢ Hardcodierte Secrets in Java/YAML
- Niemals `APP_PASSWORD`, `FTP_PASSWORD` als Literal ins Code
- Nur via `${ENV_VAR}` laden
